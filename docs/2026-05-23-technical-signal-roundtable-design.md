# 阶段三：技术指标信号层设计文档

## 概述

阶段三（技术指标信号层）是圆桌辩论系统的**纯数字输出层**，负责将前两阶段的定性判断和定量审计转化为具体的交易参数。该层不参与基本面讨论，不审计陈述，只基于价格/成交量数据输出技术信号和数值参数。

## 定位与职责

- **输入**：当前股价、成交量、历史OHLCV数据
- **输出**：信号方向（看涨/看跌/中性）、关键价位（支撑/突破/目标/止损）、信号强度
- **角色**：11位技术指标创始人，各自独立计算，仅在信号冲突时进行内部辩论
- **不参与**：阶段一的定性辩论、阶段二的定量审计、基本面判断

## 执行流程

### 步骤1：数据准备
- 获取当前标的的OHLCV数据（时间范围：日线+周线）
- 计算各指标所需的基础数据（移动平均、波动率等）
- 数据源：yfinance（美股/港股）、akshare（A股）

### 步骤2：独立输出（11人并行）
- 每位大师基于自身指标框架计算信号
- 输出标准模板：
  ```
  ### [大师姓名] 信号
  
  **信号方向**：[看涨/看跌/中性]
  **信号强度**：[0-100，基于指标置信度]
  
  **关键价位（定义）**
  - 支撑位：$[价格]——指标框架下的底部支撑区，价格跌至此处指标发出反弹信号
  - 突破位：$[价格]——趋势确认的临界点，价格突破此处确认方向信号
  - 目标价区间：$[下限] - $[上限]——基于指标测量法的预期目标范围
  - 止损参考：$[价格]——指标框架下的技术止损位（如跌破支撑位）
  
  **指标状态**
  - [指标1]：[状态，如"MACD金叉，柱状图转正"]
  - [指标2]：[状态，如"RSI 65，未超买"]
  
  **适用条件**：[该指标在何种市场环境下最有效]
  ```

### 步骤3：冲突检测
系统扫描11位大师的信号方向：
- 标记所有"看涨"大师和"看跌"大师
- 如果存在至少1对相反信号（如Appel看涨 vs Lane看跌），触发辩论
- 中性信号不参与辩论

### 步骤4：定向辩论（仅冲突对参与）
- 辩论规则：
  - 1-2轮交锋
  - 焦点："为什么当前市场环境下你的框架更可靠？"
  - 不强制达成一致，分歧保留
- 示例辩论：
  ```
  **辩论：MACD (Appel) 看涨 vs KDJ (Lane) 看跌**
  
  **Appel**：当前MACD金叉，柱状图转正，显示趋势动量向上。KDJ在震荡市中易产生假信号。
  
  **Lane**：KDJ %K已进入超买区（>80），显示短期过热。MACD滞后，可能错过顶部。
  
  **Appel**：趋势市应优先趋势指标，MACD确认了中期方向。
  
  **Lane**：同意趋势判断，但短期回调风险高，建议等待KDJ回落。
  ```

### 步骤5：投票统计
- 按信号方向统计票数：
  ```
  看涨：X票
  看跌：Y票  
  中性：Z票
  ```
- 加权计算（采用信号强度加权）：
  - 权重 = 信号强度（0-100，归一化为0-1）
  - 加权看涨分 = Σ(看涨票的信号强度)
  - 加权看跌分 = Σ(看跌票的信号强度)
  - 净方向 = (加权看涨分 - 加权看跌分) / (加权看涨分 + 加权看跌分 + 中性票强度)
  - 分歧度 = 1 - |净方向|（0-1，越高分歧越大）

### 步骤6：最终输出
阶段三输出必须包含以下三部分：

#### 1. 信号矩阵（表格）
| 大师 | 信号方向 | 信号强度 | 支撑位 | 突破位 | 目标价 | 止损 | 关键状态 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|----------|
| Appel | 看涨 | 75 | $125.00 | $138.50 | $145-$150 | $120.00 | MACD金叉 |
| Lane | 看跌 | 65 | $130.00 | $128.00 | $140-$145 | $135.00 | KDJ超买 |
| ... | ... | ... | ... | ... | ... | ... | ... |

#### 2. 投票统计
```
## 投票统计

| 方向 | 票数 | 加权分 | 占比 |
|------|:---:|:---:|:---:|
| 看涨 | 7 | 520 | 58% |
| 看跌 | 3 | 280 | 32% |
| 中性 | 1 | 90 | 10% |

净方向：+0.26（偏看涨）
分歧度：0.48（中等分歧）
```

#### 3. 操作建议（基于共识）
- **入场条件**：突破$[突破位]且成交量放大
- **止损位**：$[综合止损参考]（取各大师止损的加权平均）
- **目标位**：$[目标区间下限] - $[上限]
- **仓位建议**：基于信号强度和分歧度（高共识→标准仓位，高分歧→减半仓位）

## 与阶段二的关系

### 串行执行
```
阶段一（定性辩论） → 阶段二（技术审计） → 阶段三（技术信号）
```

### 数据依赖
- 阶段三**不依赖**阶段二的审计结果（技术指标只认价格数据）
- 阶段三的输出**独立于**阶段二的数据可信度评级
- 最终报告合并时，阶段三的操作建议会**标注**："技术信号基于价格数据，若阶段二审计发现数据问题，请谨慎参考"

### 输出合并
最终报告结构：
```
## 综合报告

### 定性结论（阶段一）
[大师圆桌核心发现]

### 技术审计（阶段二）
[审计矩阵 + 可信度评级]

### 技术信号（阶段三）
[信号矩阵 + 投票统计 + 操作建议]

### 最终操作指引
整合阶段三的操作建议 + 阶段二的数据可信度标注
```

## 质量门

| # | 检查项 | 判定标准 |
|---|--------|----------|
| 1 | 是否11位大师都输出了信号方向+数值参数？ | 11份信号报告完整，无缺失 |
| 2 | 信号冲突时是否触发了辩论？ | 存在相反信号时至少触发1对辩论 |
| 3 | 输出是否包含信号矩阵+投票统计+操作建议三部分？ | 三部分完整，格式正确 |
| 4 | 操作建议是否基于共识（而非单一大师）？ | 建议基于加权净方向和分歧度，非单一大师信号 |
| 5 | 信号强度是否明确定义？ | 基于指标数值的置信度（0-100），如RSI超买程度、MACD背离幅度等 |

## 执行纪律

1. **不越界**：技术指标层不评价基本面，不说"这个公司好/坏"
2. **不篡改**：信号方向如实呈现，不因多数票而改变少数派信号
3. **不合并**：辩论不强制达成一致，分歧本身是重要信息
4. **不重复**：与阶段二的江恩/利弗莫尔不重复（阶段二顾问层输出时间周期/趋势情绪，阶段三输出具体价位）

## 信号强度定义

信号强度（0-100）是各大师对自身指标所发信号的置信度评分，**不是外部权重**而是指标自身特性：

| 强度区间 | 标签 | 含义 |
|:---:|:---:|------|
| 80-100 | 强信号 | 指标出现教科书级标准信号（如MACD完美金叉、RSI极度超卖、布林带极端收窄后突破） |
| 60-79 | 中等偏强 | 指标信号明确但有轻微瑕疵（如MACD金叉但柱状图尚未转正） |
| 40-59 | 中性偏弱 | 指标信号模糊，接近临界区（如RSI接近50、ADX低于20） |
| 20-39 | 弱信号 | 指标勉强发出信号，需要其他指标确认 |
| 0-19 | 极弱/无效 | 指标未形成有效信号，或信号与当前市场条件矛盾 |

**强度计算原则**：由各大师 Skill 内部根据指标数值与阈值的关系自行评定，不跨指标统一。例如：
- Appel：MACD的DIFF-DEA差值和柱状图斜率共同决定强度
- Lane：Stochastic的%K位置（超买超卖区得更强，中区得更弱）
- Wilder (RSI)：RSI偏离50的幅度越大，信号越强
- Bollinger：带外突破 + 带宽变化确认，信号较强

## 数据层架构

### 数据调度编排Skill（复用）
使用之前设计的混合数据源策略：
- yfinance（美股/港股）
- akshare（A股）
- 缓存机制（1小时TTL）

### 预计算 vs 自行计算
| 指标类型 | 计算方式 | 理由 |
|----------|----------|------|
| 标准指标（MA/RSI/MACD/KDJ） | 预计算 | 多位大师需要，避免重复计算 |
| 专有指标（TD序列/OBV/W%R） | 大师Skill自行计算 | 算法特殊，需要大师专属逻辑 |
| 多时间框架 | 预计算基础数据 | 提供原始数据，大师自行转换 |

### 数据分发
为每位大师准备定制化数据包（由数据调度编排Skill统一分发）：
- **Appel (MACD)**：OHLCV + 预计算EMA(12)/EMA(26)/MACD基础数据
- **Bollinger (布林带)**：OHLCV + 预计算MA(20)/标准差(20)
- **Granville (MA+OBV)**：OHLCV + 预计算MA系列（5/10/20/60/120） + 成交量数据
- **Hosoda (一目均衡)**：OHLCV + 预计算转换线(9)/基线(26)/先行跨度A/B
- **Wilder (RSI/DMI/ADX/ATR/PSAR)**：OHLCV + 预计算RSI(14)/DMI基础数据/ATR(14)/PSAR基础数据
- **Lane (Stochastic/KDJ)**：OHLCV + 预计算%K(14,3)/%D(3)
- **Lambert (CCI)**：OHLCV + 预计算CCI(20)基础数据
- **L. Williams (W%R)**：OHLCV + 预计算W%R(14)基础数据
- **Chaikin (CMF)**：OHLCV + 预计算AD线/CMF(20)基础数据
- **Elder (三重滤网)**：多时间框架OHLCV（周线/日线/4小时） + 预计算趋势/动量指标基础数据
- **B. Williams (混沌操作法)**：OHLCV + 预计算鳄鱼线（5/8/13）/分形/AC/AO基础数据
